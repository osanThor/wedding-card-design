# 모바일 청첩장 서비스 기획

## 목표

디자인 경험이 없어도 예비 신랑, 신부가 10분 안에 예쁜 모바일 청첩장을 만들고 공유할 수 있게 한다.

단순한 정보 전달형 모바일 청첩장을 넘어, 하객이 직접 참여하고 반응을 남길 수 있는 초대 경험 플랫폼으로 확장한다.

## 핵심 사용자

- 제작자: 청첩장을 직접 만들고 사진, 지도, 계좌, 방명록을 관리한다.
- 하객: 링크로 접속해 예식 정보를 확인하고 메시지 작성, 계좌 복사, 지도 열기, 참석 여부 전달, 게임 참여, 축하 콘텐츠 남기기를 한다.

## 서비스 컨셉 확장

기존 모바일 청첩장이 예식 정보를 보여주는 정적인 페이지라면, 이 서비스는 제작자와 하객이 함께 완성하는 인터랙티브 초대장이다.

핵심 방향:

- 제작자는 청첩장 형태를 고르는 것이 아니라 초대 경험 타입을 고른다.
- 하객은 보는 사람에서 참여하는 사람으로 바뀐다.
- 예식 전, 예식 당일, 예식 후까지 이어지는 기록을 만든다.

## 초대 경험 타입

### 클래식 청첩장

가장 기본적인 모바일 청첩장이다.

- 메인 이미지
- 인사말
- 예식 정보
- 갤러리
- 지도
- 계좌번호
- 방명록
- 공유

### 게임식 청첩장

하객이 간단한 게임이나 미션을 완료하며 신랑, 신부의 이야기를 확인하는 타입이다.

예시:

- 둘의 첫 만남 맞히기
- 신랑 신부 밸런스 게임
- 연애 타임라인 카드 뒤집기
- 결혼식 D-day 미션
- 축하 메시지를 남기면 다음 사진이 열리는 구조
- 정답을 맞히면 숨겨진 감사 메시지 또는 쿠폰 이미지 공개

목표:

- 하객 체류 시간을 늘린다.
- 공유하고 싶은 재미 요소를 만든다.
- 신랑, 신부의 스토리를 자연스럽게 전달한다.

### 여행권 청첩장

청첩장을 여행 티켓, 탑승권, 초대권처럼 보여주는 타입이다.

예시:

- Boarding Pass 디자인
- Destination: Wedding Day
- Gate: 예식장
- Departure: 예식 시간
- Seat: 하객 이름 또는 랜덤 좌석 코드
- QR 초대권
- 지도와 오시는 길을 여행 일정처럼 표현

목표:

- 일반 청첩장과 시각적으로 차별화한다.
- 공유 이미지, OG 이미지, QR 카드로 확장하기 좋다.
- 여행, 공항, 티켓 감성의 프리미엄 템플릿으로 수익화할 수 있다.

### 스토리북 청첩장

신랑, 신부의 이야기를 챕터 방식으로 읽어 내려가는 타입이다.

예시:

- Chapter 1. 처음 만난 날
- Chapter 2. 함께한 계절
- Chapter 3. 결혼을 약속한 날
- Chapter 4. 초대의 말
- 마지막 챕터에서 RSVP, 지도, 계좌, 방명록 노출

목표:

- 감성적이고 몰입감 있는 청첩장을 만든다.
- 사진과 문장이 많은 제작자에게 적합하다.

### 하객 참여형 롤링페이퍼

청첩장 공개 후 하객들이 함께 축하 페이지를 채우는 타입이다.

예시:

- 축하 메시지 카드
- 이모지 리액션
- 사진 한 장 남기기
- 음성 축하 메시지
- 결혼식 전까지 열리는 카운트다운 메시지
- 신랑, 신부가 고정하거나 숨길 수 있는 메시지 관리

목표:

- 방명록을 단순 텍스트에서 참여형 콘텐츠로 확장한다.
- 예식 후에도 추억 아카이브로 남길 수 있다.

## 제작자 플로우

1. 간편 로그인
2. 새 청첩장 만들기
3. 템플릿 선택
4. 기본 정보 입력
5. 섹션 구성 선택
6. 사진과 음악 등록
7. 모바일 미리보기
8. 링크 생성
9. 카카오톡, 문자, 링크 공유

## 하객 인터랙션 플로우

1. 청첩장 링크 접속
2. 초대 경험 타입에 맞는 첫 화면 진입
3. 예식 정보 확인
4. 선택형 인터랙션 참여
5. 축하 메시지, 리액션, 사진, 음성 중 하나 이상 남김
6. 지도, 계좌, RSVP 등 실용 기능 사용
7. 카카오톡 또는 링크로 다른 하객에게 공유

## 하객 인터랙션 기능

### 가벼운 참여

- 이모지 축하 리액션
- 참석 응원 버튼
- 신랑 측, 신부 측 응원 카운트
- 축하 스티커 남기기
- 한 줄 방명록

### 콘텐츠 참여

- 사진 업로드
- 음성 축하 메시지
- 짧은 영상 축하 메시지
- 하객 질문 답변
- 예식 당일 사진 공유

### 게임 참여

- 커플 퀴즈
- 밸런스 게임
- 추억 카드 뒤집기
- 축하 미션
- 랜덤 축하 문구 뽑기

### 실용 참여

- RSVP
- 동행 인원 입력
- 식사 여부 입력
- 셔틀버스 탑승 여부
- 축의금 계좌 복사
- 지도 앱 열기

- 노션처럼 자유로운 섹션 구성

- 감성 템플릿과 섬세한 애니메이션
- 게임식 초대 경험
- 여행권, 스토리북 같은 비정형 청첩장 템플릿
- 하객이 함께 완성하는 롤링페이퍼형 청첩장
- 예식 후에도 남는 참여 기록

## 우선순위 제안

초기에는 모든 인터랙션을 만들기보다, 제작 난이도와 차별화 효과를 기준으로 작게 시작한다.

1. 여행권 템플릿
2. 이모지 리액션
3. 밸런스 게임
4. 하객 축하 카드
5. 하객 참여 현황

이 순서가 좋은 이유:

- 여행권 템플릿은 시각적 차별화가 크고 기능 복잡도는 낮다.
- 이모지 리액션은 DB 구조가 단순하지만 하객 참여감을 만든다.
- 밸런스 게임은 공유성과 재미가 높다.
- 축하 카드는 기존 방명록을 자연스럽게 확장한다.
- 참여 현황은 제작자에게 서비스 재방문 이유를 만든다.
